# Documentação dos Endpoints ## Rotas Home || api/ ### GET / Retorna a página inicial da API ## Rotas de Usuários || api/users ### Rotas: - GET /: Recupera todos os usuários. - GET /id/:id: Recupera um usuário pelo ID. - POST /: Cria um novo usuário. - PUT /id/:id: Atualiza um usuário existente pelo ID. - DELETE /id/:id: Deleta um usuário pelo ID. - POST /login: Autentica um usuário. Cada rota usa o UserControllerFactory para realizar as operações necessárias. ### Tratamento de Erros: - Retorna código de status 500 com a mensagem 'Erro Interno do Servidor' para quaisquer erros do servidor. - Retorna código de status 404 com a mensagem 'Usuário não encontrado' se o usuário não for encontrado. - Retorna código de status 401 com a mensagem 'Não autorizado' para falhas de autenticação. ### Exemplo de uso: - GET /api/users/id/1 Retorna o usuário com o ID 1. ## Rotas de Clientes || api/clients ### Rotas: - GET /: Recupera todos os clientes. - GET /id/:id: Recupera um cliente pelo ID. - POST /email: Recupera um cliente pelo email. - POST /: Cria um novo cliente. - PUT /id/:id: Atualiza um cliente existente pelo ID. - DELETE /id/:id: Deleta um cliente pelo ID. Cada rota usa o ClientControllerFactory para realizar as operações necessárias. ### Tratamento de Erros: - Retorna código de status 500 com a mensagem 'Erro Interno do Servidor' para quaisquer erros do servidor. - Retorna código de status 404 com a mensagem 'Cliente não encontrado' se o cliente não for encontrado. ### Exemplo de uso: - GET /api/clients/id/1 Retorna o cliente com o ID 1. ## Rotas de Transações || api/transactions ### Rotas: - GET /: Recupera todas as transações. - GET /id/:id: Recupera uma transação pelo ID. - POST /: Cria uma nova transação. - PUT /id/:id: Atualiza uma transação existente pelo ID. - GET /client/:id: Recupera transações pelo ID do cliente. - DELETE /id/:id: Deleta uma transação pelo ID. Cada rota usa o TransactionControllerFactory para realizar as operações necessárias. ### Tratamento de Erros: - Retorna código de status 500 com a mensagem 'Erro Interno do Servidor' para quaisquer erros do servidor. - Retorna código de status 404 com a mensagem 'Transação não encontrada' se a transação não for encontrada. ### Exemplo de uso: - GET /api/transactions/id/1 Retorna a transação com o ID 1. ## Rotas de Relatórios || api/reports ### Rotas: - GET /: Recupera todos os relatórios. - GET /id/:id: Recupera um relatório pelo ID. - POST /: Cria um novo relatório. - PUT /id/:id: Atualiza um relatório existente pelo ID. - GET /client/:id: Recupera relatórios pelo ID do cliente. - DELETE /id/:id: Deleta um relatório pelo ID.
