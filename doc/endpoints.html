# Documentação dos Endpoints ## Rotas Home || api/ ### GET http://localhost:5015/ Retorna a página inicial da API ## Rotas de Usuários || api/users ### Rotas: - GET http://localhost:5015/api/user/: Recupera todos os usuários. - GET http://localhost:5015/api/users/id/:id: Recupera um usuário pelo ID. - GET http://localhost:5015/api/users/username/:username: Recupera um usuário pelo username. POST http://localhost:5015/api/user/login HTTP/1.1 content-type: application/json { "username": "admin", "password": "admin" } POST http://localhost:5015/api/user HTTP/1.1 content-type: application/json { "username": "userP", "password": "userP", "permission": "usuario", "clientId": 1 } - PUT http://localhost:5015/api/users/id/:id: Atualiza um usuário existente pelo ID. - DELETE http://localhost:5015/api/users/id/:id: Deleta um usuário pelo ID. - POST http://localhost:5015/api/users/login: Autentica um usuário. Cada rota usa o UserControllerFactory para realizar as operações necessárias. ### Tratamento de Erros: - Retorna código de status 500 com a mensagem 'Erro Interno do Servidor' para quaisquer erros do servidor. - Retorna código de status 404 com a mensagem 'Usuário não encontrado' se o usuário não for encontrado. - Retorna código de status 401 com a mensagem 'Não autorizado' para falhas de autenticação. ### Exemplo de uso: - GET http://localhost:5015/api/users/id/1 Retorna o usuário com o ID 1. ## Rotas de Clientes || api/clients ### Rotas: - GET http://localhost:5015/api/clients: Recupera todos os clientes. - GET http://localhost:5015/api/clients/id/:id: Recupera um cliente pelo ID. - POST http://localhost:5015/api/clients/email: Recupera um cliente pelo email. - POST http://localhost:5015/api/clients: Cria um novo cliente. - PUT http://localhost:5015/api/clients/id/:id: Atualiza um cliente existente pelo ID. - DELETE http://localhost:5015/api/clients/id/:id: Deleta um cliente pelo ID.
